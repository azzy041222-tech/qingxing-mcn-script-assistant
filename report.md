# 轻醒小红书达人调研与商单脚本方案报告

## 1. 项目背景

品牌「轻醒」是一款 0 蔗糖高蛋白希腊酸奶，口味包含原味、蓝莓、黄桃。产品希望在小红书短视频平台进行自然种草，目标人群为 22-35 岁城市女性，关注健身、控糖、轻食和上班族效率生活。

本次方案目标不是生成泛泛广告文案，而是模拟 MCN 商单脚本交付流程：先调研适合的达人，再拆解内容风格，最后产出可拍摄、可质检、可写入飞书文档的脚本和分镜。

## 2. Brief 拆解

### 品牌与产品

- 品牌：轻醒
- 产品：0 蔗糖高蛋白希腊酸奶
- 口味：原味、蓝莓、黄桃

### 核心卖点

- 高蛋白
- 饱腹感
- 低负担
- 适合早餐、运动后、下午茶

### 目标人群

- 22-35 岁城市女性
- 关注健身、控糖、轻食
- 重视上班族效率生活
- 偏好自然、真实、非硬广的内容表达

### 合规限制

- 不能承诺减肥效果
- 不能承诺治疗效果
- 不能承诺降糖效果
- 避免夸大功效
- 脚本必须适合真实达人拍摄

## 4. 最终选择达人

### 首选达人：凌二七

- 人设匹配：她的「早餐搭子」「不折磨自己的健康饮食」与轻醒希望传递的轻食、早餐和运动后加餐场景高度一致。
- 内容场景匹配：早餐盘、酸奶碗、运动后补给、下午茶都能自然出现酸奶，不需要强行硬广。
- 表达方式匹配：适合第一人称分享，用「我最近这样搭」的口吻降低广告感。
- 品牌卖点匹配：高蛋白、0 蔗糖、希腊酸奶、低负担可以作为产品属性进入脚本。
- 合规可控性：把表达收束为产品属性和个人食用体验，避免减肥、降糖、代餐等承诺。

### 备选达人：文静不pang

- 人设匹配：早餐、brunch、烘焙和轻食内容更偏精致生活方式，适合视觉化呈现酸奶。
- 内容场景匹配：春日早餐盘、brunch、甜品杯等场景适合做轻醒三口味组合。
- 表达方式匹配：适合温暖治愈的日常分享和教程式内容。
- 品牌卖点匹配：0 蔗糖、高蛋白、低负担可以作为搭配食材说明出现。
- 合规可控性：避免「减脂餐」「控糖友好」等容易被理解为功效承诺的表达。

## 5. 参考内容拆解

### 可参考的内容结构

```text
生活痛点/场景切入 -> 真实选择理由 -> 产品自然出现 -> 使用/食用过程 -> 个人感受 -> 轻 CTA
```

### 可参考的开头钩子

- 上班太赶的时候，我会怎么准备早餐？
- 健身后不想乱吃，我一般会选这种轻负担加餐。
- 下午 4 点嘴馋但不想点奶茶，可以这样吃。

### 可参考的镜头语言

- 冰箱取出酸奶
- 展示口味和质地
- 加水果、坚果或燕麦
- 办公桌/健身包/早餐桌自然出现
- 第一人称试吃
- 结尾展示完整食用场景

## 6. AI 工作流设计

1. Brief 拆解 Agent：提取产品信息、目标人群、卖点和限制。
2. 达人风格分析 Agent：分析博主人设、语气、镜头、内容结构。
3. 脚本生成 Agent：基于 brief 与达人风格输出短视频脚本。
4. 风险质检 Agent：检查功效承诺、夸大表达和平台风险。
5. 飞书写入 Agent：将最终脚本、分镜和质检结果写入飞书文档。

## 9. 复盘

### 做得好的地方

- 工作流完整覆盖 brief 拆解、达人风格分析、脚本生成、合规质检和飞书写入。
- 最终脚本围绕真实早餐和运动后加餐场景展开，产品植入较自然。
- 风险质检明确区分禁用表达、谨慎表达和可用表达。

### 遇到的问题

- 小红书站内内容访问受登录和反爬限制影响，部分数据需要结合人工核验。
- 飞书写入涉及应用权限、文档授权和本地运行环境，需要在 README 中保留可复现配置说明。
- 中文文件名和 PowerShell 默认编码在部分终端下容易出现乱码，需要用 UTF-8 查看。

### 后续可优化

- 补充真实合作前的产品包装营养数据核验。
- 如果进入实际投放，可增加不同达人版本脚本和 A/B 标题。
- 将飞书写入命令进一步封装成一条固定脚本，降低复现门槛。
