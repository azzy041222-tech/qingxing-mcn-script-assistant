# MCN Script Assistant

## 适用场景

当需要为小红书、抖音、视频号等内容平台生成 MCN 商单短视频脚本时使用。尤其适用于食品、轻食、饮品、生活方式、健身加餐、办公室场景类产品。

本 Skill 适合完成以下任务：

- 品牌 brief 拆解
- 达人内容风格分析
- 商单脚本生成
- 分镜设计
- 合规风险检查
- 飞书文档写入前的结构化整理

## 输入材料

使用前尽量准备以下信息：

1. 品牌 brief
   - 品牌名
   - 产品名
   - 核心卖点
   - 目标人群
   - 投放平台
   - 禁用表达和合规限制

2. 达人信息
   - 达人主页链接或截图
   - 内容方向
   - 粉丝画像判断
   - 近期 2-3 条代表内容
   - 可参考的内容结构、钩子、镜头、表达方式

3. 交付要求
   - 视频时长
   - 脚本格式
   - 分镜数量
   - 是否需要飞书文档格式

## 执行步骤

### Step 1 - Brief 拆解 Agent

提取品牌、产品、卖点、人群、场景、限制和风险表达。

输出格式：

```text
品牌：
产品：
核心卖点：
目标人群：
适合场景：
禁用表达：
内容建议：
```

### Step 2 - 内容风格模仿 Agent

分析达人是否适合该 brief，重点看：

- 人设是否可信
- 内容场景是否自然
- 粉丝画像是否匹配
- 是否适合软性产品植入
- 是否存在合规或调性风险

只模仿结构，不照搬文案。

可参考维度：

- 开头钩子类型
- 视频节奏
- 场景切换
- 第一人称表达方式
- 产品出现位置
- 结尾互动方式

### Step 3 - 脚本生成 Agent

生成 60-90 秒短视频脚本，必须包含：

- 标题
- 开头钩子
- 完整口播
- 产品植入点
- 结尾 CTA
- 拍摄注意事项
- 合规风险提醒

### Step 4 - 风险质检 Agent

检查脚本是否出现以下风险：

- 承诺减肥、瘦身、掉秤
- 承诺治疗或改善疾病
- 承诺降糖、控血糖效果
- 使用绝对化表达
- 夸大高蛋白、饱腹、低负担效果
- 语气过度硬广
- 不适合真实达人拍摄

如果发现风险，必须给出：

- 风险句子
- 风险原因
- 修改建议
- 修改后可用表达

### Step 5 - 飞书文档写入 Agent

将最终内容整理成飞书文档结构，至少包含：

- 达人信息
- 内容策略
- 视频标题
- 完整脚本
- 分镜表格
- 产品植入点
- CTA
- 合规风险提醒
- 质检结论

如有飞书 API 权限，调用自动化脚本写入飞书文档；如没有权限，记录权限限制、排查过程和降级方案。

### Step 6 - 分镜生成

至少输出 6 个镜头，每个镜头包含：

- 镜头编号
- 画面
- 口播/字幕
- 道具/场景
- 时长建议

## 输出格式

```markdown
# 商单脚本方案

## 1. 达人选择理由

## 2. 内容策略

## 3. 视频标题

## 4. 开头钩子

## 5. 完整口播脚本

## 6. 分镜设计

| 镜头 | 画面 | 口播/字幕 | 道具/场景 | 时长 |
| --- | --- | --- | --- | --- |

## 7. 产品植入点

## 8. 结尾 CTA

## 9. 合规风险提醒

## 10. 质检结果
```

## 风险检查清单

- [ ] 是否避免“减肥”“瘦身”“掉秤”等效果承诺？
- [ ] 是否避免“治疗”“改善疾病”等医疗化表达？
- [ ] 是否避免“降糖”“控血糖”等功效承诺？
- [ ] 是否避免“最”“一定”“完全不会胖”等绝对化表达？
- [ ] 是否将“高蛋白”“饱腹感”“低负担”表达为体验或产品属性，而非效果保证？
- [ ] 是否符合达人原有内容风格？
- [ ] 是否能被真实达人低成本拍摄？
- [ ] 是否有自然的产品植入点？
- [ ] 是否有清晰但不过度销售化的 CTA？

## 示例任务

```text
请基于以下 brief 和达人风格，生成一条 60-90 秒小红书短视频商单脚本，并附上 6 个以上分镜和合规质检结果。

Brief：
{{brief}}

达人风格：
{{influencer_style}}

参考内容拆解：
{{content_breakdown}}
```
