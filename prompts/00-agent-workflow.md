# 轻醒 MCN 脚本生成 Agent 工作流

## 工作流目标

将品牌 brief、小红书博主调研结果和内容拆解，转化为一条适合真实达人拍摄的小红书短视频商单脚本，并完成风险质检和飞书文档写入。

## 输入材料

1. 品牌 brief
   - 品牌：轻醒
   - 产品：0 蔗糖高蛋白希腊酸奶
   - 口味：原味、蓝莓、黄桃
   - 卖点：高蛋白、饱腹感、低负担
   - 场景：早餐、运动后、下午茶
   - 人群：22-35 岁城市女性
   - 限制：不能承诺减肥、治疗、降糖等功效

2. 博主调研
   - 首选：凌二七
   - 次选：文静不pang
   - 代表内容、标题结构、镜头语言、适配理由和风险点

3. 输出要求
   - 60-90 秒小红书短视频脚本
   - 至少 6 个分镜
   - 合规风险质检结果
   - 飞书文档写入格式

## 5 步流程 Agent

### Step 1 - Brief 拆解 Agent

**目标**：把客户 brief 拆成脚本创作需要的结构化信息。

**输入**：

- 客户 brief

**输出**：

- 产品信息
- 目标人群
- 核心卖点
- 推荐内容场景
- 禁用表达
- 脚本方向建议

**关键约束**：

- 不把食品写成有减肥、治疗、降糖功效的产品。
- 卖点表达以产品属性和食用体验为主。

### Step 2 - 内容风格模仿 Agent

**目标**：基于博主调研结果，提炼可借鉴的内容结构，而不是照搬原文。

**输入**：

- 博主信息卡
- 代表内容标题
- 人设、场景、表达方式、镜头拆解

**输出**：

- 博主人设关键词
- 标题公式
- 开头钩子类型
- 视频结构
- 画面调性
- 语言风格
- 可模仿点和不可照搬点

**关键约束**：

- 只学习结构、节奏和场景，不复制博主原文。
- 保留达人原有语气，但让产品植入自然出现。

### Step 3 - 脚本生成 Agent

**目标**：生成 1 条 60-90 秒小红书短视频商单脚本。

**输入**：

- Brief 拆解结果
- 博主风格分析
- 内容拆解
- 合规限制

**输出**：

- 标题
- 开头钩子
- 完整口播文案
- 产品植入点
- 结尾 CTA
- 拍摄提示

**关键约束**：

- 产品不能一上来硬广。
- 先建立生活场景，再自然出现轻醒酸奶。
- 优先写“早餐搭配”或“运动后加餐”，不写功效承诺。

### Step 4 - 风险质检 Agent

**目标**：检查脚本中可能违反食品类广告或平台内容规范的表达。

**输入**：

- 脚本初稿

**输出**：

- 总体结论：通过 / 需修改 / 高风险
- 风险句子
- 风险原因
- 修改建议
- 修改后版本

**质检规则**：

- 禁止：减肥、瘦身、掉秤、降糖、降脂、治疗、改善疾病、替代正餐。
- 谨慎：控糖友好、必备、一定要试试、完全无负担。
- 可用：0 蔗糖、高蛋白、低负担、饱腹感、早餐搭配、运动后补给。

### Step 5 - 飞书写入 Agent

**目标**：将最终脚本、分镜和质检结果整理成飞书文档格式，并通过 API 或自动化脚本写入飞书。

**输入**：

- 最终脚本
- 分镜设计
- 风险质检结果

**输出**：

- 飞书文档结构化内容
- 飞书写入脚本执行结果
- 飞书文档链接或截图

**关键约束**：

- 不能只手动复制粘贴。
- 如遇 API 权限限制，需要记录排查过程和降级方案。

## 流程兜底机制

- 如果博主信息不足，先生成“需人工补充清单”，不虚构粉丝数据。
- 如果脚本风险过高，返回脚本生成 Agent 重写。
- 如果飞书 API 无权限，保留脚本、报错记录、截图和接入说明。
