### 完整口播文案

大家好，我是凌二七。

如果你早上总是来不及吃饭，或者运动后很想吃点甜的，今天这 3 个酸奶搭配可以直接抄作业。

我最近常吃的是轻醒 0 蔗糖高蛋白希腊酸奶。它的质地比普通酸奶更厚一点，舀起来是比较扎实的厚酸奶感。

第一碗是蓝莓燕麦酸奶碗。蓝莓味酸奶打底，加即食燕麦、蓝莓、坚果，再撒一点奇亚籽。这个搭配不用开火，早上十分钟就能做好。

第二个是黄桃酸奶吐司杯。把吐司切成小块垫底，铺上黄桃味希腊酸奶，再加黄桃丁和一点椰子片。吃起来有点像不用烤箱的小甜品，下午茶也很合适。

第三个是原味酸奶咸口碗。原味希腊酸奶加一点黑胡椒和柠檬汁，旁边放水煮蛋、番茄、牛油果和全麦贝果。这个更适合运动后，想吃得扎实一点但又不想太油的时候。

我觉得希腊酸奶最方便的一点，就是甜口咸口都能搭。想吃甜的就配水果，想吃正餐感就配鸡蛋和全麦面包。

如果你也在找早餐、运动后或者下午茶的低负担搭子，可以试试轻醒这三个口味。原味更百搭，蓝莓和黄桃更适合直接做甜口酸奶碗。

今天这 3 个搭配我会放在字幕里，大家可以按自己冰箱里有的食材替换。记得好好吃饭，不要折磨自己。

## 九、飞书自动化接入方案

### 推荐方案：飞书开放平台 API 写入

适用场景：稳定复用、适合脚本批量生成、无需人工复制粘贴。

实际已验证脚本：

```powershell
$env:FEISHU_APP_ID="<app_id>"
$env:FEISHU_APP_SECRET="<app_secret>"
node .\feishu_write_direct.js
```

流程：

1. 读取本地 Markdown 脚本文档。
2. 使用 `tenant_access_token` 调用飞书开放平台。
3. 创建 Docx 文档。
4. 将 Markdown 拆分为飞书 blocks 写入。
5. 返回飞书文档链接和 blocks 写入数量。

优点：

- 不依赖手动复制粘贴。
- 可在 PowerShell 中直接运行。
- 适合后续自动化批量生成文档。

注意：

- App 需要具备 `docx:document:create` 与文档写入权限。
- App Secret 不应写入 README 或代码文件，应通过环境变量临时注入。

### 备选方案：lark-cli 用户授权写入

适用场景：需要以用户身份创建文档，或飞书 App 权限不足时。

示例命令：

```powershell
cmd.exe /c "cd /d C:\Users\31963\Desktop && lark-cli docs +create --title \"轻醒x凌二七短视频脚本\" --markdown @轻醒_凌二七_短视频脚本.md"
```

流程：

1. 安装 `lark-cli`。
2. 执行 `lark-cli auth login --recommend` 扫码授权。
3. 将 Markdown 文件放到目标目录。
4. 使用 `docs +create --markdown @file` 自动创建飞书文档。

注意：

- `--markdown @file` 建议使用相对路径，先 `cd` 到文件所在目录。
- User token 有有效期，过期后需重新扫码授权。
- 在 Windows/WSL 混合环境中，优先用 `cmd.exe /c` 调用。
